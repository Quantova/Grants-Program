> [program process](../docs/process.md) for how to submit.

- **Team Name:** Your legal entity or team name.
- **Payment Address:** The QTOV address your grant will be paid to. Please also
  specify the currency (QTOV).

## Project Overview

If this application is in response to an RFP, please indicate this on the first line
of this section. If this is a follow-up to an earlier successful Quantova grant,
please provide the name and/or pull request of that grant on the first line.

- **Tagline:** If your project name isn't descriptive, a one-sentence summary.
- **Description:** A brief description of your project.
- **Relation to Quantova:** Explain how your project is related to Quantova — the
  protocol, the QVM, QRC20, QNS, bridges, the post-quantum stack, or developer
  tooling. Be specific.
- **Motivation:** Why are you building this? What problem does it solve for the
  Quantova ecosystem?

### Project Details

We expect this section to contain, where relevant:

- Mockups, designs, or wireframes for any user-facing components.
- Data models / API specifications of the core functionality.
- An overview of the technology stack to be used.
- Documentation of core components, protocols, architecture, etc. to be designed
  and/or built.
- A PoC, demo, or any prior work, if you have one.

- What need(s) does your project meet?
- Are there other projects similar to yours in the Quantova ecosystem? If so, how
  is yours different? If not, are there similar projects in other ecosystems?

- **Estimated Duration:** e.g. 3 months
