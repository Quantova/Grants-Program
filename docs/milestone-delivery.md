# Milestone Delivery

Once your grant is approved and you've completed a milestone, submit it for
evaluation and payment using the process below.

## How to deliver a milestone

1. Make sure each deliverable listed in your application's milestone table is
   complete and meets its specification (license, documentation, tests, article,
   and the features for that milestone).
2. Open a **milestone delivery** pull request (or issue) in this repository that
   includes:
   - a link to your accepted application,
   - the milestone number,
   - links to the deliverables (repository, commit hashes, docs, tests, article),
   - clear instructions for how the committee can run and verify the work.
3. A committee member (and, where useful, an evaluator) reviews the delivery
   against the specification.

If you cannot use a pull request, email **Grants@quantova.org** with the same
information.

## What the evaluators check

- Each deliverable matches the specification in the approved application.
- Code is open-source under the agreed license.
- Documentation lets a user run and use the deliverable.
- Tests exist and the testing guide explains how to run them.
- The article/tutorial (where required) is published.

## Revisions

If something is incomplete or doesn't match the specification, the evaluators will
request changes in the delivery PR. Address them and update the PR.

## Payment

When the milestone is approved, the corresponding QTOV payment is disbursed to the
payment address in your application. Repeat for each milestone until the grant is
complete.
